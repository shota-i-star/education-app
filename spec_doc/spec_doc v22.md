## 1. プロジェクト概要

| 項目 | 内容 |
|------|------|
| アプリ名 | AZism 教育進捗管理 |
| 種類 | Webアプリ（ブラウザで動作）・PWA対応（ホーム画面追加可） |
| 対象ユーザー | 全社員（MG・メンバー・管理者） |
| 開発者のスキル | プログラミング未経験。作りながら学ぶスタイルで進める |

## 2. 解決したい課題

- 誰がどの教育項目をどこまで終えたか、一目でわからない
- テストの点数や進捗を手作業（Excel等）で管理しており、集計が煩雑
- 上司とメンバーで見せる情報を分けたい

## 3. 主な機能要件

### メンバー向け画面
- チェックリスト形式で教育項目を完了マークできる
- テストを受けて点数が記録される（**カテゴリ×等級ごと**にテストが設定されている場合のみボタン表示）
- **テストボタンを押すと問題集一覧モーダルが表示され、受けるテストを選択できる（v20〜）**
- **受験済みテストは結果閲覧のみ可能（回答編集不可・v20〜）**
- **採点待ちのテストは「採点待ち」ステータスで表示され再受験不可（v20〜）**
- 過去の等級のコンテンツを閲覧できる
- チェック時に画面が白くフラッシュしない（silent再描画）
- 進捗を「全体進捗率」と「現在の等級の進捗率」の2軸で表示
- 進捗バーはヘッダー直下に貼り付き（sticky）、スクロールしても常時表示される
- カテゴリは管理画面の sort_order 順で表示される
- カテゴリヘッダーはスクロールすると貼り付き、次のカテゴリに到達すると自動で切り替わる
- カテゴリごとにトグル開閉が可能。▼/▶で開け閉めでき、チェック後の再描画でも開閉状態が保持される
- 等級タブはタブの変更で等級別進捗のみ切り替わり、全体進捗は固定表示

### テスト画面
- **問題集IDを URL パラメータ（?set_id=N）で受け取り、DB からテスト内容を動的に読み込む（v20〜）**
- **問題タイプ：4択（choice）/ ○×（truefalse）/ 記述（text）の3種類に対応（v20〜）**
- **合否判定：選択式の自動採点点数＋記述式の採点後点数の合計 ≥ 合格基準点 で合格（v20〜）**
- **記述式がある場合は提出後「採点中」ステータスとなり、採点完了まで合否確定しない（v20〜）**
- 受験済みのテストは過去の回答・結果を表示（回答編集不可）
- 採点中・受験済みの場合は再受験不可（二重送信ガード付き）

### MG・管理者向け画面
- 全メンバーの進捗を一覧・集計して見られる
- 進捗率は「全体進捗率」「現等級進捗率」の2軸で表示
- サマリーカードに「メンバー数」「平均進捗率（全体）」「平均進捗率（現等級）」「テスト平均点」「未合格者数」の5指標を表示
- 進捗率セクションは「エリア別」「店舗別」「等級別」「個人別」の4タブ
- 等級別タブ：0〜3等級ごとにメンバーをグループ化し、展開でメンバー一覧表示
- エリア別・店舗別タブで行をクリックするとその配下のメンバー一覧が展開表示される
- テストの平均点・個人別スコアを確認できる
- 進捗率をエリア別・店舗別グラフで把握できる
- メンバー詳細画面でチェック状況・テスト履歴・コメントを確認できる
- **メンバー詳細のテスト履歴：問題集名・合計点（選択式＋記述式）・採点中ステータスを表示（v20〜）**
- メンバー詳細画面からPDFで出力できる

### 管理者向け画面
- メンバー管理（追加・部署変更・等級変更・役割変更・削除）
- メンバー編集モーダル：名前・社員番号・パスワードを1画面で変更
- メンバー追加時に部署の設定が必須
- 社員番号変更・パスワードリセット（全権管理者のみ）
- メンバーデータのCSVエクスポート
- CSV一括インポート：メンバー・カテゴリ・店舗・項目
- **CSVテンプレートファイルのヘッダー行とモーダル説明文を完全一致させた（v22〜）**
- 部署・エリア・店舗管理（追加・削除・トグル表示・店舗のエリア間移動）
- **部署エリア店舗管理の店舗移動・削除ボタンを右寄せ表示（v20〜）**
- 事業部名・エリア名の編集（全権管理者のみ）
- カテゴリ管理（追加・複製・削除・CSV一括追加・ドラッグ＆ドロップ並べ替え）
- カテゴリ編集モーダル：カテゴリ名の編集に加え、配下の項目をドラッグ＆ドロップで並び替え可能
- 項目管理（追加・No.表示・項目名編集・複製・削除・CSV一括追加）
- テスト管理（部署×等級ごとに整理・トグル表示）
- **問題集追加：まず部署を選択 → その部署に紐づくカテゴリのみ選択可能（v20〜）**
- **問題集の編集：問題集名・カテゴリ・等級・合格基準点を後から変更可能（v21〜）**
- **問題タイプ：4択・○×・記述の3種類。○×は管理画面で「○が正解」「×が正解」を設定（v20〜）**
- **問題の編集：問題文・形式・配点・選択肢・正解を後から変更可能（v21〜）**
- 記述式テストの採点（各問題の設定配点を最大点として入力）
- **採点入力欄：問題に設定された配点を最大点として表示（v20〜）**
- 再試験管理（合計点が合格基準未満かつ採点完了済みのみ対象）
- **テスト管理画面の問題一覧：形式・配点・編集・削除ボタンを右端に固定表示（v21〜）**
- **テスト結果ページ：問題集ごとに全メンバーの受講状況・点数・回答内容を確認できる（v22〜）**
- カテゴリ複製機能（manager_admin のみ）

### 共通機能
- ログイン機能（役割に応じて画面を切り替え）
- ログアウト機能（全画面共通）
- データはSupabaseに保存（リロードしても消えない）
- スマートフォン対応（member_top.html・test_page.html）
- PWA対応
- AZ-GROUPロゴを全ページヘッダーに表示
- ヘッダーの「AZism 教育進捗管理」タイトルをタップすると画面が更新される

### その他テーブル（v19と同様）

### 合否の計算方法（v20〜）

```
合計点 = test_results.score（選択式・○×の自動採点）
         + SUM(text_answers.score)（その test_set_id の記述式採点済み合計）

合否 = 合計点 >= test_sets.pass_score

採点中 = text_answers に score=null の行が1件でもある場合
         → 合否未確定。表示上「採点中」とし、最終合否は採点完了後に再計算
```

## 7. 各バージョンで追加・修正した主な機能

### v20 で追加・修正した機能
（詳細は spec_doc v20.md / handover_v20.md を参照）

- テスト画面の全面刷新（動的読み込み・3種問題タイプ・合否ロジック・再受験ブロック）
- メンバー画面のテスト選択モーダル・採点待ちブロック
- 管理画面の問題集追加フロー改善・採点配点表示・再試験管理修正・店舗ボタン右寄せ
- メンバー詳細のテスト履歴改善（問題集名・合計点・採点中ステータス）
- CSVテンプレート修正（メンバーのエリア列削除・店舗の列順変更）

### v21 で追加・修正した機能
（詳細は spec_doc v21.md / handover_v21.md を参照）

1. **問題集の編集機能**：問題集カードの「編集」ボタン → 編集モーダルで問題集名・部署・カテゴリ・等級・合格基準点を変更可能
2. **問題の編集機能**：問題行の「編集」ボタン → 編集モーダルで問題文・形式・配点・選択肢・正解を変更可能
3. **問題一覧のレイアウト修正**：問題文セルに `width:100%` を付与して可変幅とし、形式・配点・編集・削除の4列を常にテーブル右端に固定表示

### v22 で追加・修正した機能

#### 管理画面（admin.html）
1. **テスト結果ページの新規追加**：サイドバー「採点」セクションに「テスト結果」を追加。問題集ごとのトグルカードで全メンバーの受講状況・点数・回答内容を確認できる
2. **部署フィルタ**：`manager_admin` は全部署・全問題集が対象。`dept_admin` は `category_departments` 経由で自部署カテゴリの問題集のみ表示。メンバーも `allowedMemberIds` で自部署に限定

#### CSVテンプレート（csv/）
3. **メンバーCSVテンプレートのヘッダー修正**：`csv_template_members.csv` の1行目を、モーダル説明文と完全一致させた（半角括弧 → 全角括弧、スラッシュ前後にスペース追加）

## 8. 役割一覧
| role | 表示名 | 権限 |
|---|---|---|
| member | メンバー | 自分の学習画面のみ |
| manager | MG | 全部署のダッシュボード閲覧 |
| manager_admin | 全権管理者 | 全部署閲覧＋全管理機能＋社員番号変更・PW変更・部署/エリア名編集 |
| dept_admin | 部署管理者 | 自部署のみ閲覧・管理（メンバー名前編集可） |

※ `admin` ロールは廃止済み

## 9. ログイン後の遷移先

```javascript
if (role === 'member') → member_top.html
if (role === 'manager' || role === 'dept_admin') → manager_dashboard.html
if (role === 'manager_admin') → admin.html
```
