## 2. 解決したい課題

- 誰がどの教育項目をどこまで終えたか、一目でわからない
- テストの点数や進捗を手作業（Excel等）で管理しており、集計が煩雑
- 上司とメンバーで見せる情報を分けたい

---

## 3. 主な機能要件

### メンバー向け画面
- チェックリスト形式で教育項目を完了マークできる
- テストを受けて点数が記録される（**カテゴリ×等級ごと**にテストが設定されている場合のみボタン表示・v19〜）
- 過去の等級のコンテンツを閲覧できる
- チェック時に画面が白くフラッシュしない（silent再描画）
- 進捗を「全体進捗率」と「現在の等級の進捗率」の2軸で表示
- **進捗バーはヘッダー直下に貼り付き（sticky）、スクロールしても常時表示される（v19〜）**
- **カテゴリは管理画面の sort_order 順で表示される（v19〜）**
- **カテゴリヘッダーはスクロールすると貼り付き、次のカテゴリに到達すると自動で切り替わる（v19〜）**
- **カテゴリごとにトグル開閉が可能。▼/▶で開け閉めでき、チェック後の再描画でも開閉状態が保持される（v19〜）**
- 等級タブはタブの変更で等級別進捗のみ切り替わり、全体進捗は固定表示

### MG・管理者向け画面
- 全メンバーの進捗を一覧・集計して見られる
- 進捗率は「全体進捗率」「現等級進捗率」の2軸で表示
- サマリーカードに「メンバー数」「平均進捗率（全体）」「平均進捗率（現等級）」「テスト平均点」「未合格者数」の5指標を表示
- 進捗率セクションは「エリア別」「店舗別」「等級別」「個人別」の4タブ
- 等級別タブ：0〜3等級ごとにメンバーをグループ化し、展開でメンバー一覧表示
- エリア別・店舗別タブで行をクリックするとその配下のメンバー一覧が展開表示される
- テストの平均点・個人別スコアを確認できる
- 進捗率をエリア別・店舗別グラフで把握できる
- メンバー詳細画面でチェック状況・テスト履歴・コメントを確認できる
- メンバー詳細画面からPDFで出力できる

### 管理者向け画面
- メンバー管理（追加・部署変更・等級変更・役割変更・削除）
- メンバー編集モーダル：名前・社員番号・パスワードを1画面で変更
- メンバー追加時に部署の設定が必須
- 社員番号変更・パスワードリセット（全権管理者のみ）
- メンバーデータのCSVエクスポート
- CSV一括インポート：メンバー・カテゴリ・店舗・項目
- 部署・エリア・店舗管理（追加・削除・トグル表示・店舗のエリア間移動）
- 事業部名・エリア名の編集（全権管理者のみ）
- カテゴリ管理（追加・複製・削除・CSV一括追加・ドラッグ＆ドロップ並べ替え）
- **カテゴリ編集モーダル：カテゴリ名の編集に加え、配下の項目をドラッグ＆ドロップで並び替え可能（v19〜）**
- 項目管理（追加・No.表示・項目名編集・複製・削除・CSV一括追加）
- **項目の並び順は管理画面／メンバー画面の両方に反映される sort_order で制御（v19〜）**
- **項目削除は外部キー制約を考慮し、check_progress → check_steps → check_items の順に削除（v19〜）**
- テスト管理（部署×等級ごとに整理・トグル表示）
- 記述式テストの採点
- 再試験管理
- カテゴリ複製機能（manager_admin のみ）

### 共通機能
- ログイン機能（役割に応じて画面を切り替え）
- ログアウト機能（全画面共通）
- データはSupabaseに保存（リロードしても消えない）
- スマートフォン対応（member_top.html・test_page.html）
- PWA対応
- AZ-GROUPロゴを全ページヘッダーに表示
- ヘッダーの「AZism 教育進捗管理」タイトルをタップすると画面が更新される

### check_steps / check_progress / test_sets 等は v18 と同様。

### 進捗率の計算方法

**現等級進捗率**
```
現等級進捗率 = チェック済みstep数（現在の等級のみ）÷ 現在等級の全step数 × 100
```

**全体進捗率**
```
全体進捗率 = チェック済みstep数 ÷ 部署の全カテゴリ・全等級のstep数 × 100
```

## 7. v19 で追加・修正した主な機能

### メンバー画面（member_top.html）
1. カテゴリの並び順を管理画面と同じ **sort_order 順** に統一
2. 進捗バーをヘッダー直下に **sticky** で常時表示（スクロールしても見える）
3. カテゴリヘッダーを **sticky 化**。スクロールで現在のカテゴリヘッダーが進捗バー下に貼り付き、次のカテゴリに到達すると次のヘッダーに切り替わる
4. テストボタンの表示判定を **カテゴリ×等級** ごとに変更（該当カテゴリにテストが未作成なら非表示）
5. カテゴリごとに **▼/▶ トグル開閉** 機能を追加（チェック後の再描画でも開閉状態が保持される）

### 管理画面（admin.html）
1. 項目行の D&D を廃止し、**カテゴリ編集モーダル内** で項目を並び替える方式に変更
2. 並び替え制約：同じカテゴリ＋同じ等級内のみ
3. 並び替え後は No. が自動で振り直される
4. **項目削除の不具合修正**：check_progress → check_steps → check_items の順に削除、`adminClient`（service key）で RLS を回避
